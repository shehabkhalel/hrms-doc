HRMS : Project Structure

Created by shehab ahmed khalil, last modified on Jun 07, 2023
Components

  • This directory is for shared components that are used inside more than one page (main route).

  • Each component has to be inside a folder with the name of the component (capitalized) that contains a main js file (index.js).

  • The component could be built using other shared components, or child components.

  • Unique child components must be placed inside the component folder

  • All components must follow the best practices and the right conventions as provided in the general guidelines page.

Hooks

  • This directory is for custom hooks in the app

  • All hooks must follow the best practices and the right conventions as provided in the general guidelines page.

Pages

  • This directory is for the main pages in the app and not for small components.

  • Each page should have it’s own directory (capitalized) inside /pages directory and a JS file named index.js (main file)

  • A page could have dedicated child or nested components, and they should be placed in the same directory as the index.js file as long as they aren’t shared with other pages.

Services

  • This directory is for services that will be integrated in different pages

  • Each service should be created in a separate directory that contains a file with the service name (Fields.service.js)

  • Each service will contain different services (getFields, getField, updateField, deleteField, ..etc) and all these services should be exported

  • These services will use react-query utils and functions to retrieve and modify data.

Utils

  • This directory is for helper functions and utilities that will be used across the app.

Internationalization

  • This directory is dedicated for internationalization

  • Each page should have a json file that will contain en, ar and any other language code objects, these objects will contain the keys and values for translations in this page. 

    {
  "en": {
    "login.title": "Login",
    "login.username": "Username",
    "login.password": "Password",
    "login.button": "Login"
  }
}

  • After creating a file for the page, this file should be imported inside the i18n file as follows 

      resources: {
    en: {
      translation: {
        ...login.en,
      },
    },
  },
