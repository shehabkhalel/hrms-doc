HRMS : General guidelines

Created by shehab ahmed khalil, last modified on Jun 07, 2023

  • A component shouldn’t ever contain un-used imports or variable (keep the code clean and the terminal free of annoying warnings)

  • All components in the project should have the same order of importing modules as follows: React modules → React-dom modules → Material UI components → custom hooks → child components → icons and images → css modules

  • All handlers functions should be wrapped in a callback function following react best practices for optimizations

  • All presentational components should be exported memo-ized for optimization (to prevent unnecessary re-renders)

  • Always write documentations for components and functions describing the parameters and the return value and purpose of use.

  • Components design should follow SOLID principles (Component should have single responsibility and should be scalable) https://www.freecodecamp.org/news/solid-principles-explained-in-plain-english/
