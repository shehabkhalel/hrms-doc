Smart-dumb components

Smart dumb components design pattern is simply having two types of components:

Smart:

  • handles all the logic, fetching and manipulating data and state management.

  • It doesn’t contain jsx or styles

  • It wraps the dump components and passes them props and methods as needed

Dumb:

  • Dumb components are just presentational components (UI).

  • They get passed the data and just simply show these data with the styles.

  • They doesn’t handle any heavy logic.

This approach helps with separate of concerns, scalability, code maintenance and easy to test.

Examle:

Smart

import { useEffect, useState } from "react";
import Dumb from "./Dumb";

/**
 * @returns
 */
const Smart = () => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    new Promise((resolve) => setTimeout(resolve, 1000)).then(() => {
      setLoading(false);
      setUser({ name: "shehab", age: 25 });
    });
  }, []);

  return <Dumb user={user} loading={loading} />;
};
export default Smart;

Dumb

/**
 * @param {Object} user
 * @param {Boolean} loading
 * @returns
 */
const Dumb = ({ user, loading }) => {
  if (loading) return <div class="loading">Loading...</div>;
  else {
    return (
      <div>
        <p class="name">name: {user.name}</p>
        <p class="age">age: {user.age}</p>
      </div>
    );
  }
};
export default Dumb;
